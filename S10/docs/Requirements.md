# Requirements List

1. The system must allow students to submit a service request.
2. Each request must contain a title.
3. Each request must contain a description.
4. Each request must have a unique request ID.
5. Each request must have a status.
6. The default status of a new request must be Pending.
7. The system must allow users to view a list of requests.
8. The system must allow users to view the details of a specific request.
9. Staff members must be able to update the request status.
10. The allowed statuses must be Pending, In Progress, and Done.
11. The system must validate required input fields before saving data.
12. The system should keep data, business logic, and interface separated.
13. The system should use clear class and method names.
14. The system should be easy to maintain and extend in the future.
15. The system should follow a simple MVC structure.
